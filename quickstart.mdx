# Quickstart

Get a working Prompt Tips API running locally in under 5 minutes. This guide walks you through installation, seeding sample data, and making your first API calls.

## Prerequisites

- Node.js 16 or higher
- npm or yarn

## Setup

<Steps>
  <Step title="Install dependencies">
    Clone the repository and install packages:

    ```bash
    git clone https://github.com/asifshaikat/prompt-tips.git
    cd prompt-tips
    npm install
    ```
  </Step>

  <Step title="Configure environment">
    Create your `.env` file from the example:

    ```bash
    cp .env.example .env
    ```

    The defaults work for local development:

    ```bash .env
    PORT=3000
    DATA_DIR=./data
    CACHE_MAX_AGE=60
    STALE_WHILE_REVALIDATE=300
    MAX_JSON_BODY=4096
    ```
  </Step>

  <Step title="Seed sample data">
    Load the sample tips from the YAML seed file:

    ```bash
    npm run seed
    ```

    This creates per-tip files from `data/seeds/tips.seed.yaml`:

    ```
    Seeded 4 tips.
    ```
  </Step>

  <Step title="Build global view">
    Generate the double-buffered global index:

    ```bash
    npm run rebuild
    ```

    You should see:

    ```
    Rebuilt global view → b tips: 4
    ```
  </Step>

  <Step title="Start the server">
    Launch the API server:

    <CodeGroup>
    ```bash npm
    npm run dev
    ```

    ```bash direct
    NODE_ENV=development node src/server.js
    ```
    </CodeGroup>

    The server starts on port 3000:

    ```
    [tips] listening on :3000
    ```
  </Step>
</Steps>

## Try it out

### List all tips (hot sorting)

```bash
curl http://localhost:3000/api/tips
```

<CodeGroup>
```json Response
{
  "items": [
    {
      "id": 1,
      "slug": "be-explicit-about-output-format",
      "title": "Be explicit about output format",
      "content": "Ask for JSON with a schema or a bullet list with exact fields. Models follow structure better than vague requests.",
      "tags": ["format", "json"],
      "status": "published",
      "created_at": "2025-01-01T09:00:00Z",
      "updated_at": "2025-01-01T09:00:00Z",
      "love_count": 0,
      "meh_count": 0
    }
  ],
  "nextCursor": "2025-01-01T09:00:00Z_1",
  "cachedAt": "2026-03-03T15:30:00.000Z"
}
```
</CodeGroup>

### Get a specific tip

```bash
curl http://localhost:3000/api/tips/1
```

### Cast a vote

Vote with `1` (love), `0` (neutral), or `-1` (meh):

```bash
curl -X POST http://localhost:3000/api/tips/1/vote \
  -H "Content-Type: application/json" \
  -d '{"vote": 1, "userToken": "u_demo_user_123"}'
```

<CodeGroup>
```json Response
{
  "id": 1,
  "love_count": 1,
  "meh_count": 0
}
```
</CodeGroup>

<Info>
The `userToken` is a client-generated identifier. The same token can change votes but can't vote twice on the same tip.
</Info>

### Pagination and filtering

Fetch the next page with the cursor:

```bash
curl "http://localhost:3000/api/tips?sort=hot&limit=2&cursor=2025-01-01T09:00:00Z_1"
```

Filter by time window:

<CodeGroup>
```bash 24 hours
curl "http://localhost:3000/api/tips?window=24h"
```

```bash 7 days
curl "http://localhost:3000/api/tips?window=7d"
```

```bash All time
curl "http://localhost:3000/api/tips?window=all"
```
</CodeGroup>

Change sort strategy:

<CodeGroup>
```bash Hot (default)
curl "http://localhost:3000/api/tips?sort=hot"
```

```bash New
curl "http://localhost:3000/api/tips?sort=new"
```

```bash Top
curl "http://localhost:3000/api/tips?sort=top"
```
</CodeGroup>

## Test caching with ETags

The API returns ETags for cache validation. Make the same request twice:

```bash
# First request - full response
curl -i http://localhost:3000/api/tips

# Copy the ETag from the response header
# Example: ETag: "v-a1b2c3d4e5f6"

# Second request with If-None-Match
curl -i http://localhost:3000/api/tips \
  -H 'If-None-Match: "v-a1b2c3d4e5f6"'
```

The second request returns `304 Not Modified` with no body, saving bandwidth.

Open http://localhost:3000 to see the simple web interface with voting buttons and sort tabs.
